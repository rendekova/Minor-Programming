CSS Property Descriptions (Experimentation Results)

Box Model Properties

  • **`padding`:** Space **inside** the box (between the content and the border). *Observation:* When changed to `40px`, the background area of the box grew significantly, increasing the space around the text, but the text size itself did not change. [Image of the CSS Box Model]
  • **`margin`:** Space **outside** the box (between the border and other elements). *Observation:* When changed to `30px`, the distance separating the boxes from each other and from the container edge increased dramatically.
  • **`height`:** Sets the explicit vertical dimension of the content area. *Observation:* When set to `50px`, the content space inside the box shrank, making the boxes appear shorter overall.
  • **`border`:** The line around the content and padding. *Observation:* Used `2px solid black` to clearly define the box boundary.

Positioning Properties

  • **`position: static;` (Box 1):** This is the **default**. The element follows the normal document flow and ignores `top/left/right/bottom` offsets.
  • **`position: relative;` (Box 2):** Positioned relative to its normal place. It moves but **still occupies its original space** in the document flow.
  • **`position: absolute;` (Box 3):** The element is **removed from the document flow**. It is positioned relative to its nearest *positioned* ancestor.
  • **`position: sticky;` (Box 4):** Acts like `relative` when scrolling until it hits the defined threshold (`top: 0`), then it "sticks" and behaves like `fixed`.
  • **`position: fixed;` (Fixed Footer):** The element is positioned relative to the **viewport** (browser window) and remains in the same spot even when the user scrolls the page.
